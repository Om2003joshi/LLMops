# 📌 Topic: Model Loader (LLMOps)

---

## 1. 🧠 Problem

In an AI system, we need:

* API keys (securely)
* Embedding model
* LLM model

But:

* Keys can come from different places (env, cloud)
* Models may change (Google, Groq, etc.)

👉 So we need a **central system** to handle this dynamically.

---

## 2. 💡 Simple Explanation

ModelLoader is like a **factory**:

* It reads config (YAML)
* It gets API keys
* It creates models (embedding + LLM)

👉 Without changing code, you can switch models.

---

## 3. 🔑 Key Concepts

* Environment Variables (`os.getenv`)
* `.env` loading (`load_dotenv`)
* YAML config (`load_config`)
* API Key Management
* Dynamic model selection
* Exception handling

---

## 4. 🧩 Components

### 🔐 ApiKeyManager

Responsible for:

* Loading API keys from:

  1. JSON (cloud secret)
  2. Environment variables
* Validating required keys
* Masking keys in logs

---

### 🤖 ModelLoader

Responsible for:

* Loading `.env` (only in local)
* Loading YAML config
* Creating:

  * Embedding model
  * LLM model

---

## 5. ⚙️ Flow (Step-by-Step)

1. Check environment (local or production)
2. Load `.env` (if local)
3. Initialize ApiKeyManager
4. Load YAML config
5. Load embedding model
6. Load LLM model based on provider

---

## 6. 📦 Embedding Model

Code:

```python
model_name = config["embedding_model"]["model_name"]
GoogleGenerativeAIEmbeddings(model=model_name, google_api_key=key)
```

👉 Converts text → vector
👉 Used in retrieval (RAG)

---

## 7. 🤖 LLM Model

### Step 1: Choose provider

```python
provider = os.getenv("LLM_PROVIDER", "google")
```

---

### Step 2: Load config

```python
llm_config = config["llm"][provider]
```

---

### Step 3: Create model

#### Google:

```python
ChatGoogleGenerativeAI(...)
```

#### Groq:

```python
ChatGroq(...)
```

---

## 8. 🔥 Key Feature

👉 Dynamic Model Switching

You can change model without touching code:

```bash
LLM_PROVIDER=google
```

or

```bash
LLM_PROVIDER=groq
```

## 10. ❌ Errors I Faced

* API key not working
* `.env` not loading
* Wrong config path

---

## 11. ✅ Fix / Learning

* Always check ENV variables
* Ensure `.env` is loaded in local
* Validate config structure
* Use logging for debugging

---

👉 ModelLoader connects:

* Config (YAML)
* Secrets (API keys)
* Models (LLM + Embeddings)

👉 It makes the system:

* Flexible
